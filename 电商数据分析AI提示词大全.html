数据清洗
基础方法
什么是数据清洗？（大白话）
就像洗菜做饭 — 买回来的菜有黄叶、泥巴、坏叶子，得先挑干净才能下锅。数据清洗就是把"脏数据"捡出来扔掉或修好，比如重复的数据、缺失的信息、写错的日期、多打的空格等。不干净的数据做出来的分析就像用烂菜做的饭，肯定不好吃！
生活比喻：就像洗衣服 — 把脏衣服洗干净，去除污渍、修补破洞、扔掉没用的。
方法思维导图
电商实际案例
淘宝订单数据清洗：某店铺导出10万条订单数据，发现有2000条重复订单（系统bug）、500条缺失手机号、300条收货地址格式混乱。清洗后保留9.7万条有效数据，分析结果准确率提升20%。
✅ 处理效果：
  • 去重复
  • 补缺失
  • 修错误
  • 统一格式
📋 使用前必看：前置条件
项目要求
📊需要数据电商原始数据（订单表、用户表、商品表等），Excel/CSV格式均可
🔧需要工具AI工具（ChatGPT、文心一言等）+ Excel或Python（可选）
🎯适用场景刚导出数据还没分析过、数据有缺失/重复、数据格式混乱
⚠️注意事项清洗前务必备份原始数据！重要字段删除前先确认
⏱️预计耗时5-10分钟（小数据量），30分钟以上（大数据量）
基础提示词
帮我清洗这份电商数据： 1. 删除完全重复的行，以及按订单号去重 2. 数值型缺失用中位数填充，文本型缺失用"未知"填充 3. 检查逻辑错误（如订单金额为负、结束时间早于开始时间） 4. 统一日期格式为YYYY-MM-DD，统一手机号格式 5. 删除完全为空的列，去除文本两端多余空格 6. 输出清洗前后的数据量对比报告

特征选择
基础方法
什么是特征选择？（大白话）
就像考试复习划重点 — 一本书有1000页，但考试只考重点章节。特征选择就是从一大堆数据字段里，挑出真正有用的"重点字段"。比如预测用户会不会买东西，用户的"浏览时长"很重要，但"用户头像颜色"就没用，把没用的去掉，分析更快更准。
生活比喻：就像收拾行李 — 只带最有用的东西，不带没用的，减轻负担走得更快。
方法思维导图
电商实际案例
京东用户流失预测：原始数据有50个字段（用户ID、注册时间、浏览时长、购买次数、收货地址、头像颜色...）。通过特征选择，挑出10个最关键的特征（最近购买时间、购买频次、消费金额等），模型训练速度提升5倍，准确率还提高了8%。
✅ 处理效果：
  • 减少维度
  • 提高速度
  • 降低过拟合
  • 提升准确率
📋 使用前必看：前置条件
项目要求
📊需要数据已清洗的电商数据集，包含目标变量（如是否购买、消费金额等）和多个特征字段
🔧需要工具AI工具（ChatGPT、文心一言等）+ Python（Pandas/sklearn，可选）
🎯适用场景特征字段太多（>20个）、模型训练慢、想提升预测准确率
⚠️注意事项业务专家的经验很重要，不要只看数据指标；保留有业务意义的特征
⏱️预计耗时10-15分钟（简单数据集），30分钟以上（高维数据）
基础提示词
帮我做电商数据的特征选择： 1. 先列出所有特征字段，计算每个特征的重要性 2. 用相关系数找出与目标变量最相关的特征 3. 删除高度相关的冗余特征（相关系数>0.8） 4. 用随机森林计算特征重要性，选出Top10 5. 对比不同特征组合的模型效果 6. 给出最终推荐的特征清单和理由

数据转换
基础方法
什么是数据转换？（大白话）
就像把食材切好腌好再下锅 — 买回来的肉是整块的，得切成片、丝、丁，还要放盐料酒腌一下。数据转换就是把原始数据"加工"成模型能看懂、好用的样子。比如把消费金额取对数（贫富差距太大）、把连续数据分成几段（年龄分段）、把多个字段组合成一个（客单价=销售额/订单数）。
生活比喻：就像做菜前的备菜 — 把食材洗好切好配好调料，炒的时候才顺手。
方法思维导图
电商实际案例
拼多多用户消费分析：用户消费金额差距极大（从10元到10万元），直接分析看不出规律。通过对数转换后，数据分布变得均匀；再把年龄分成"0-18/19-30/31-45/46+"四段，最后计算"客单价=总消费/订单数"新特征，分析效果提升明显。
✅ 处理效果：
  • 数据正态化
  • 消除量纲
  • 创造新特征
  • 提升模型效果
📋 使用前必看：前置条件
项目要求
📊需要数据已清洗的电商数据，包含数值型、类别型等多种类型的字段
🔧需要工具AI工具（ChatGPT、文心一言等）+ Excel或Python（可选）
🎯适用场景数据分布偏斜严重、字段类型不对、需要创造新的业务指标
⚠️注意事项转换后要检查是否符合业务逻辑；保留原始数据列，不要直接覆盖
⏱️预计耗时10-20分钟（简单转换），30分钟以上（复杂特征工程）
基础提示词
帮我做电商数据的转换和特征工程： 1. 先分析每个数值字段的分布情况，找出偏斜严重的字段 2. 对右偏分布的字段（如消费金额）做对数变换或开平方 3. 将连续型字段（如年龄、消费金额）按业务逻辑分箱 4. 创造新的组合特征：客单价=销售额/订单数，件单价=销售额/销量 5. 将日期字段拆解为年、月、周、星期几、是否节假日 6. 检查转换后的数据分布，输出转换前后对比报告

数据标准化
基础方法
什么是数据标准化？（大白话）
就像比赛前统一秤体重 — 举重比赛要按重量级分组，不能让100公斤的跟50公斤的比。数据标准化就是把不同"尺码"的数据放到同一个秤上。比如"浏览时长"是分钟级（0-100），"消费金额"是万元级（0-100000），两个数差了1000倍，放在一起模型会被金额"带偏"。标准化后大家都变成差不多的范围，公平比较。
生活比喻：就像考试分数换算 — 满分100和满分150的卷子，都换成百分制才能比高低。
方法思维导图
电商实际案例
淘宝用户价值模型：用"浏览时长（分钟）、收藏数（个）、消费金额（元）"三个指标预测用户价值。但消费金额（0-50000）的数值远大于其他两个，模型几乎只看金额。用Min-Max标准化后，三个指标都变成0-1之间的数，模型能公平考虑每个因素，预测准确率提升了15%。
✅ 处理效果：
  • 统一量纲
  • 公平比较
  • 加速收敛
  • 提升稳定性
📋 使用前必看：前置条件
项目要求
📊需要数据已清洗的数值型电商数据，各字段数值范围差异较大
🔧需要工具AI工具（ChatGPT、文心一言等）+ Python（sklearn/preprocessing，可选）
🎯适用场景用机器学习建模、特征间数值差异大、需要计算距离/相似度
⚠️注意事项标准化只对数值型字段做；异常值会影响Z-Score，有异常值用稳健缩放
⏱️预计耗时5-10分钟（简单标准化），15分钟以上（含异常值处理）
基础提示词
帮我做电商数据的标准化处理： 1. 先找出所有数值型字段，统计每个字段的范围（最小值、最大值、均值、标准差） 2. 检查是否有异常值，有异常值的字段用RobustScaler（稳健缩放） 3. 没有异常值的字段用Min-Max标准化缩放到[0,1]区间 4. 如果要做距离计算（如聚类），用Z-Score标准化（均值0方差1） 5. 输出标准化前后各字段的统计对比 6. 保存标准化后的数据，同时保留原始数据列

异常值处理
基础方法
什么是异常值处理？（大白话）
就像班里有个考0分或考150分的人 — 大家都考60-90分，突然冒出来一个0分或150分，这个人会把平均分"带歪"。异常值就是那些跟大多数数据差得特别远的"奇葩"数据。比如客单价都是几十几百，突然有一笔100万的订单（可能是测试单或填错了）。不处理掉这些"奇葩"，算出来的平均值、趋势线就不准了。
生活比喻：就像熬汤捞掉沫子 — 汤上面飘的那层沫子不是正经汤，得捞掉汤才好喝。
方法思维导图
电商实际案例
京东618销售分析：某店铺客单价平均200元，但有3笔订单金额超过10万元（内部测试单和企业采购单）。这3笔把客单价拉高到280元，严重失真。用箱线图法识别出这3个异常值，用盖帽法（超过99分位数的按99分位数值算）处理后，客单价回到205元，分析结果才靠谱。
✅ 处理效果：
  • 数据更准确
  • 均值更靠谱
  • 模型更稳定
  • 趋势更真实
📋 使用前必看：前置条件
项目要求
📊需要数据已清洗的电商数值型数据（销售额、订单量、客单价等）
🔧需要工具AI工具（ChatGPT、文心一言等）+ Excel或Python（可选）
🎯适用场景计算均值/中位数被极值带偏、数据分布有长尾、建模前数据预处理
⚠️注意事项异常值不一定是错的！先判断是不是真实业务（如大客户订单）；删除前备份
⏱️预计耗时5-10分钟（简单识别处理），20分钟以上（需业务确认）
基础提示词
帮我检测并处理电商数据中的异常值： 1. 对每个数值字段，用箱线图法（IQR）和3σ原则分别检测异常值 2. 统计每个字段的异常值数量和占比，列出异常值样本 3. 先判断异常值是不是真实业务数据（如大额订单、特殊活动） 4. 是错误数据的直接删除；是真实极值的用盖帽法（99%分位数）处理 5. 关键指标（如销售额）可以单独分析异常值，看看有没有业务洞察 6. 输出处理前后的数据对比（均值、中位数、标准差变化）

数据抽样
基础方法
什么是数据抽样？（大白话）
就像尝汤 — 一锅汤不可能全喝完才知道咸淡，舀一勺尝尝就知道了。数据抽样就是从一大堆数据里"挑一小撮"出来分析。比如你有1000万条订单数据，全部跑一遍要3小时，抽10万条出来分析，5分钟就搞定，而且结果差不了多少。关键是要"抽得均匀"，不能只抽VIP用户，那样就不准了。
生活比喻：就像农民验粮食 — 从一麻袋米里插个探子取一点出来，就能知道整袋米的质量。
方法思维导图
电商实际案例
淘宝双11用户调研：某品牌有500万购买用户，想做用户满意度调研。全部问卷成本太高，用分层抽样：按消费层级（低/中/高）和年龄段分层，每层按比例抽，总共抽5000人，结果跟全量分析的误差不到1%，但成本只有原来的1%，省时又省钱。
✅ 处理效果：
  • 节省时间
  • 降低成本
  • 提高效率
  • 结果准确
📋 使用前必看：前置条件
项目要求
📊需要数据全量电商数据集（数据量较大，通常>10万条）
🔧需要工具AI工具（ChatGPT、文心一言等）+ Excel或Python（可选）
🎯适用场景数据量太大跑不动、快速验证想法、做调研问卷、测试算法效果
⚠️注意事项样本量不能太小（至少1000条）；稀有类别要用分层抽样；设置随机种子可复现
⏱️预计耗时3-5分钟（简单随机抽样），10-15分钟（分层抽样）
基础提示词
帮我从电商数据中抽样： 1. 先看一下总数据量和各关键维度的分布（用户层级、商品品类、地区等） 2. 确定抽样比例或样本量：建议样本量至少1000条，占比1%-5% 3. 如果有重要的分类维度（如用户等级、商品品类），用分层抽样保证比例一致 4. 简单情况用随机抽样，设置随机种子保证结果可复现 5. 对比抽样前后各维度的分布，确保样本有代表性 6. 输出抽样后的数据和抽样报告（样本量、各维度占比对比）

特征编码
基础方法
什么是特征编码？（大白话）
就像把中文翻译成英文给外国人听 — 电脑只认识数字，不认识"红色""大号""北京"这些文字。特征编码就是把文字型的数据翻译成数字。比如"商品颜色"有红/黄/蓝三种，不能直接放进模型里算，得把它变成1/2/3或者[1,0,0]/[0,1,0]/[0,0,1]这样的数字，模型才能看懂。
生活比喻：就像点菜报编号 — 服务员记菜名慢，你直接报编号（1号宫保鸡丁、2号鱼香肉丝），又快又不会错。
方法思维导图
电商实际案例
京东商品分类预测：数据里有"商品颜色"（红/黄/蓝/白/黑5种）、"商品尺码"（S/M/L/XL 4种）、"所属品类"（50个品类）。颜色和尺码用独热编码变成01向量；品类有50种，独热编码太散，用目标编码（按每个品类的平均转化率来编码），效果好维度还低。
✅ 处理效果：
  • 文本转数字
  • 模型可用
  • 保留信息
  • 控制维度
📋 使用前必看：前置条件
项目要求
📊需要数据包含类别型字段的电商数据（如品类、颜色、尺码、地区、用户等级等）
🔧需要工具AI工具（ChatGPT、文心一言等）+ Python（Pandas/sklearn，可选）
🎯适用场景数据里有文字型分类字段、要做机器学习建模、类别数量多或少的情况
⚠️注意事项有大小顺序的用序数编码（如小/中/大），没有的用独热编码；高基数类别用目标编码
⏱️预计耗时5-10分钟（简单编码），15-20分钟（含高基数处理）
基础提示词
帮我做电商数据的特征编码： 1. 先找出所有类别型字段，统计每个字段有多少个不同取值 2. 有大小顺序的类别（如用户等级、尺码）用序数编码（1/2/3...） 3. 没有大小关系且类别少的（<10个）用独热编码（One-Hot） 4. 类别特别多的高基数字段（如商品ID、城市）用目标编码 5. 日期字段提取年、月、日、星期几作为新特征 6. 输出编码前后的字段对比和维度变化

数据增强
基础方法
什么是数据增强？（大白话）
就像菜不够，加点配菜凑一盘 — 数据太少不够分析或训练模型，就"造"一些类似的数据出来。数据增强就是在真实数据基础上，生成更多"差不多但又不一样"的数据。比如你只有100个流失用户样本，模型学不好，就基于这100个稍微改动一下（加一点噪声、微调数值），变成500个，模型就能学得更好了。
生活比喻：就像翻花样做菜 — 一块鸡肉，可以做宫保鸡丁、辣子鸡、鸡汤，本质都是鸡肉，但做法不同花样就多了。
方法思维导图
电商实际案例
拼多多欺诈订单检测：某店铺10万订单里只有200个欺诈订单，正负样本比例1:500，模型根本学不会识别欺诈。用SMOTE算法对欺诈样本做数据增强，合成800个类似的欺诈样本，比例变成1:100，模型召回率从30%提升到75%，能多识别出好几倍的欺诈订单。
✅ 处理效果：
  • 样本更多
  • 模型更准
  • 解决不平衡
  • 提升泛化
📋 使用前必看：前置条件
项目要求
📊需要数据样本量较少或类别不平衡的电商数据（如欺诈检测、流失预测）
🔧需要工具AI工具（ChatGPT、文心一言等）+ Python（imblearn/sklearn，可选）
🎯适用场景正负样本严重不平衡、数据量太少模型学不好、想提升模型泛化能力
⚠️注意事项只能在训练集上做增强，测试集不能动；增强后的数据不能偏离真实业务太多
⏱️预计耗时10-15分钟（简单过采样），20-30分钟（SMOTE/复杂方法）
基础提示词
帮我做电商数据的数据增强： 1. 先看一下目标变量的分布，统计各类别的样本量和比例 2. 如果是类别不平衡问题（如欺诈/流失），对少数类做过采样 3. 样本量少的用SMOTE算法合成新样本，不要简单复制 4. 数值型特征可以加一点小噪声（<5%）来增加多样性 5. 注意：数据增强只能在训练集上做，测试集保持原样 6. 输出增强前后的样本分布对比，以及增强方法说明

文本数据清洗
基础方法
什么是文本数据清洗？（大白话）
就像整理用户评论 — 评论里有表情符号、脏话、打错的字、重复的话、还有广告水军。文本数据清洗就是把这些乱七八糟的文字整理干净。比如商品评价里的"这件衣衣好好看啊啊啊😘😘😘"，得先去掉表情、去掉重复字、统一简体字，变成"这件衣服好好看"，后面才能做情感分析、关键词提取。
生活比喻：就像编辑审稿子 — 把错别字改了、废话删了、语气理顺，最后变成通顺的文章。

时间序列处理
基础方法
什么是时间序列处理？（大白话）
就像看天气预报 — 不是看某一天的温度，而是看连续很多天的温度变化，才能预测明天冷不冷。时间序列处理就是把按时间排好的数据整理好，看出规律来。比如每天的销售额、每小时的访客数，数据是按时间顺序来的，要处理掉节假日的异常、填补缺失的日期、分解出趋势和周期，后面才能做预测。
生活比喻：就像看股票K线图 — 把每天的价格按时间连起来，才能看出是涨是跌、有没有规律。
方法思维导图
电商实际案例
京东销售预测：某店铺有过去2年的日销售额数据，但缺了十几天的数据（系统升级），而且双11、618这些大促数据特别异常。先把缺失日期用线性插值补上，再用时间序列分解，分离出长期上升趋势、每周的周期规律、和节假日影响，最后做预测准确率比直接用原始数据高了30%。
✅ 处理效果：
  • 时间连续
  • 发现规律
  • 分离趋势
  • 方便预测
📋 使用前必看：前置条件
项目要求
📊需要数据带时间戳的电商数据（日/周/月销售额、访客数、订单量等）
🔧需要工具AI工具（ChatGPT、文心一言等）+ Python（Pandas/statsmodels，可选）
🎯适用场景销售预测、流量预测、库存预测、季节性分析、趋势分析
⚠️注意事项时间序列数据顺序不能乱；大促/节假日数据要单独标记，不能当普通数据
⏱️预计耗时10-15分钟（简单处理），20-30分钟（含季节分解和特征提取）
基础提示词
帮我处理电商时间序列数据： 1. 先检查时间是否连续，有没有缺失的日期，缺失的用线性插值补上 2. 统一时间粒度：按日/周/月汇总（根据分析需求选择） 3. 用时间序列分解，分离出趋势项、季节项（周/月/年）、残差项 4. 提取时间特征：年、月、周、星期几、是否周末、是否节假日、是否大促 5. 计算移动平均（7天/30天）和滞后特征（前1天/前7天/同比） 6. 输出可视化建议和各成分的分析结论

转化率概率
基础概念
什么是转化率概率？（大白话）
100个人进你店里，有几个人会买东西？这个"几个人/100人"就是转化率概率。比如3%的转化率，意思就是平均每100个访客里有3个人会下单。就像抛硬币，正面朝上的概率是50%，意思是抛很多次的话，大概一半是正面。转化率概率就是"来了多少人"和"买了多少人"的比值。
生活比喻：就像钓鱼 — 撒了100竿，有3竿钓到鱼，那"上鱼率"就是3%，这就是转化率概率。
概念思维导图
电商实际案例
淘宝直通车优化：某女装店铺直通车一天有10000次点击，其中300人下单购买，转化率就是3%。行业平均是2.5%，说明这家店的详情页做得不错。又发现首页进来的转化率是5%，搜索进来的是2%，说明首页流量更精准，可以加大首页投放。
✅ 核心作用：
  • 衡量效果
  • 对比渠道
  • 优化页面
  • 预测销售
📋 理解前必看：前置条件
项目要求
📊需要数据访客数、点击数、下单数、支付数等基础数据，两个数就能算转化率
🔧需要工具计算器就行，Excel更方便；AI可以帮你分析转化率高低的原因
🎯适用场景评估广告效果、优化详情页、对比不同渠道、做销售预测、选品
⚠️注意事项样本量太少时转化率不准（比如10个访客2个下单，不能说20%转化率）；要看长期趋势
⏱️预计耗时2分钟就能算出来，分析原因10-15分钟
基础提示词
帮我分析电商转化率数据： 1. 先算一下各个环节的转化率：点击率、加购率、下单率、支付率 2. 对比不同渠道（搜索/首页/直通车/直播）的转化率差异 3. 看看转化率高的渠道有什么共同点，低的原因可能是什么 4. 按商品品类、价格带、用户层级分别算转化率，找规律 5. 跟行业平均转化率对比，看看我们处于什么水平 6. 给出提升转化率的3-5个具体建议

贝叶斯概率（用户行为预测）
基础概念
什么是贝叶斯概率？（大白话）
就像侦探破案 — 一开始你觉得嫌疑人A有50%概率是凶手，后来发现A有作案时间、有动机、有凶器，你就觉得他90%是凶手了。贝叶斯概率就是"先有个大概估计，然后根据新证据不断更新判断"。比如用户刚进店，你觉得他买东西的概率是3%；后来他浏览了5个商品、加购了2个，你就更新成30%；他又领了优惠券，就更新成60%。
生活比喻：就像看天气预报 — 早上说70%概率下雨，中午乌云密布了，就变成90%；下午出太阳了，就变成10%。
概念思维导图
电商实际案例
京东用户购买预测：某用户刚进店铺，根据历史数据先验购买概率是3%（100人里3个买）。然后发现他浏览了3个商品（浏览的人里40%会买），更新为15%；又加购了1个商品（加购的人里60%会买），更新为40%；还领了优惠券（领券的人里70%会买），更新为65%。这时候就该给他发个提醒，促他下单。
✅ 核心作用：
  • 动态预测
  • 精准营销
  • 流失预警
  • 个性化推荐
📋 理解前必看：前置条件
项目要求
📊需要数据用户行为数据（浏览/加购/收藏/领券）和历史转化率，各行为对应的购买概率
🔧需要工具AI工具（ChatGPT可以帮你算）+ Excel或Python（复杂情况用）
🎯适用场景用户购买概率预测、流失预警、精准营销时机判断、个性化推荐
⚠️注意事项先验概率要准，不然结果差很多；行为之间可能有关联，不能简单相乘
⏱️预计耗时理解概念10分钟，实际应用（有数据）15-30分钟
基础提示词
用贝叶斯概率帮我预测用户购买概率： 1. 先确定先验概率：整体用户的购买转化率是多少 2. 列出关键行为及其对应的购买概率（浏览/加购/收藏/领券/看评价等） 3. 假设一个用户的行为路径，一步步更新他的购买概率 4. 找出购买概率最高的前3个行为，也就是"最能预示购买的信号" 5. 基于这些信号，设计一套用户分层策略（高/中/低购买意向） 6. 给不同层级的用户推荐不同的运营动作（发券/提醒/推荐商品）

泊松分布（订单量预测）
基础概念
什么是泊松分布？（大白话）
你开了家店，平均每天卖100单，但具体每天卖多少单呢？可能90单，也可能110单，偶尔130单。泊松分布就是用来算"平均每天固定来多少，但具体哪天来多少是随机的"这种事的概率。比如平均每天100单，那明天来80单的概率是多少？来120单的概率是多少？用泊松分布一算就知道。
生活比喻：就像等公交 — 平均10分钟来一辆，但具体等多久不一定，可能2分钟也可能20分钟，泊松分布能算各种情况的概率。
概念思维导图
电商实际案例
拼多多618库存准备：某爆款商品大促期间平均每小时卖50单。用泊松分布一算：每小时卖40-60单的概率约68%，卖30-70单的概率约95%，卖少于20单或多于80单的概率很小。于是仓库按70单/小时备货，既够卖又不积压，还省了仓储成本。
✅ 核心作用：
  • 订单预测
  • 库存规划
  • 客服排班
  • 异常检测
📋 理解前必看：前置条件
项目要求
📊需要数据单位时间的订单量/访客量/咨询量等计数数据，至少有一段时间的历史数据
🔧需要工具AI可以直接帮你算和解释；Excel有POISSON函数；Python用scipy
🎯适用场景预测订单量、客服排班、库存准备、流量预估、异常订单检测
⚠️注意事项只适用于"独立事件"（每一单之间不影响）；大促期间均值会变，不能用平时的λ
⏱️预计耗时理解概念10分钟，实际计算和应用10-15分钟
基础提示词
用泊松分布帮我分析和预测电商订单量： 1. 先看一下历史每天/每小时的订单量数据，计算平均值 2. 验证一下数据是不是符合泊松分布的特点（均值约等于方差） 3. 计算不同订单量的概率：比如明天卖80单/100单/120单的概率各是多少 4. 计算95%概率下订单量会在什么范围（下限和上限） 5. 基于这个范围，给库存准备、客服排班、物流安排提建议 6. 找出订单量明显异常的日期，分析可能的原因

二项分布（优惠券效果预测）
基础概念
什么是二项分布？（大白话）
你发了1000张优惠券，每张券要么被用了，要么没被用，就两种结果。历史上优惠券使用率是20%，那这次1000张里大概多少张会被用？二项分布就是算"只有两种结果（成/不成），重复很多次，会成功多少次"的概率。比如抛10次硬币，正面朝上几次？发1000张券，多少张会被用？都用二项分布算。
生活比喻：就像投篮 — 你投篮命中率50%，投10个球，能进几个？可能5个，也可能4个或6个，二项分布能算出各种情况的概率。
概念思维导图
电商实际案例
淘宝优惠券预算：某店铺双11计划发10000张满199减20的优惠券，历史上这种券的使用率是25%。用二项分布一算：大概会用掉2500张左右，用掉2300-2700张的概率约95%。于是预算按2700张准备（2700×20=54000元），既够又不浪费，还能应对意外情况。
✅ 核心作用：
  • 预算估计
  • 效果预测
  • 风险评估
  • 活动对比
📋 理解前必看：前置条件
项目要求
📊需要数据历史活动的转化率/使用率，以及这次活动的计划数量（发多少券/多少人参加）
🔧需要工具AI可以直接算和解释；Excel有BINOM.DIST函数；Python用scipy
🎯适用场景优惠券预算、活动效果预测、投放效果预估、AB测试结果判断
⚠️注意事项每次"试验"要独立；概率p要准确，不然结果差很多；样本量越大越准
⏱️预计耗时理解概念10分钟，实际应用10-15分钟
基础提示词
用二项分布帮我分析电商活动效果： 1. 先明确问题：是什么活动？有多少人参加？历史转化率是多少？ 2. 计算预期的成功次数和范围（比如95%概率下会在什么区间） 3. 计算各种情况的概率：比如使用率低于15%的概率、高于30%的概率 4. 如果是对比两个活动（A方案和B方案），判断差异是不是真的有效 5. 基于分析结果，给这次活动的预算准备、库存准备、人员安排提建议 6. 如果想达到某个目标（比如至少2000张券被用），需要发多少张出去

条件概率（关联购买分析）
基础概念
什么是条件概率？（大白话）
普通概率是"买A的人有多少"，条件概率就是"已经买了B的人里面，有多少人也买了A"。比如100个人里有10个买了手机壳（概率10%），但买了手机的50个人里有30个也买了手机壳（条件概率60%）。这个"在买了手机的条件下，买手机壳的概率"就是条件概率，能帮我们发现商品之间的关联。
生活比喻：就像下雨天带伞 — 平时带伞的人可能只有10%，但下雨天大90%的人都会带伞，这就是"在下雨的条件下带伞的概率"。
概念思维导图
电商实际案例
京东关联推荐：某店铺分析订单数据，发现买了"游戏手柄"的人里，有45%也买了"游戏耳机"（条件概率45%）；而买了"洗面奶"的人里，只有5%买了"游戏耳机"。于是把游戏耳机和游戏手柄捆绑销售，详情页放"买了又买"推荐，游戏耳机的销量直接涨了30%。
✅ 核心作用：
  • 发现关联
  • 搭配推荐
  • 捆绑销售
  • 提升客单价
📋 理解前必看：前置条件
项目要求
📊需要数据订单明细数据（每笔订单买了哪些商品），至少有几千笔订单才有意义
🔧需要工具AI可以帮你分析和解释；复杂的用Python（mlxtend/apriori算法）
🎯适用场景购物篮分析、关联推荐、捆绑销售、商品搭配、货架陈列
⚠️注意事项别把巧合当关联（比如买A的人也买了水，可能只是大家都买水）；要看提升度
⏱️预计耗时理解概念10分钟，简单分析15分钟，深度关联挖掘30分钟以上
基础提示词
用条件概率帮我做电商关联购买分析： 1. 先看一下订单数据，统计每个商品的购买概率（支持度） 2. 计算热门商品之间的条件概率：买了A的人里有多少买了B 3. 找出条件概率最高的前10组商品组合，也就是最强关联 4. 计算提升度，排除"大家都买"的假关联（比如人人都买的包邮商品） 5. 基于这些关联，设计3-5组捆绑销售方案和推荐策略 6. 给商品详情页、购物车页面的推荐位置提具体建议

正态分布（用户消费分析）
基础概念
什么是正态分布？（大白话）
就像人的身高 — 特别高和特别矮的人都很少，大部分人都在中间平均身高附近，画成图就是一个中间高、两边低的"钟形曲线"。正态分布就是"中间多、两头少"的分布规律。比如用户消费金额，大部分人花100-300元，花10块或1000块的都很少。知道了这个规律，就能快速判断"这个用户是不是正常的"，也能预测大概有多少人在什么消费水平。
生活比喻：就像考试成绩 — 大部分人考70-80分，考满分和考不及格的都很少，成绩分布就是正态分布。
概念思维导图
电商实际案例
淘宝用户消费分层：某店铺用户平均消费200元，标准差50元，符合正态分布。按正态分布规律：68%的用户消费150-250元（普通用户），95%的用户消费100-300元（绝大多数），消费超过350元的只有2.5%（高端用户）。于是店铺针对这2.5%的高端用户做VIP专属服务，投入少但效果好。
✅ 核心作用：
  • 用户分层
  • 异常检测
  • 区间估计
  • 消费预测

独立事件概率（多渠道转化）
基础概念
什么是独立事件概率？（大白话）
两个事儿互相不影响，就是独立事件。比如你今天穿什么颜色的衣服，跟明天会不会下雨，这俩没关系，就是独立的。独立事件的概率就是"两件事同时发生的概率 = 各自概率相乘"。比如从淘宝来的用户转化率是3%，从京东来的也是3%，两个渠道独立，那"淘宝来的转化了 AND 京东来的也转化了"就是3%×3%=0.09%。判断两个事件独不独立，能帮我们正确算概率。
生活比喻：就像抛两次硬币 — 第一次正面还是反面，不影响第二次的结果，两次是独立的，两次都是正面的概率就是50%×50%=25%。
概念思维导图
电商实际案例
拼多多多渠道营销：某店铺同时投抖音和快手两个渠道，抖音转化率2%，快手转化率2%，两个渠道的用户是独立的。老板想知道"一个用户同时在两个渠道都看到广告并且都转化"的概率？按独立事件算就是2%×2%=0.04%，很低。又想"至少在一个渠道转化"的概率？就是1-(1-2%)×(1-2%)=3.96%，比单个渠道高一点，但不是简单相加。

  • 避免误区

用户留存概率（复购预测）
基础概念
什么是用户留存概率？（大白话）
这个月买了东西的用户，下个月还会再来买吗？留存概率就是"用户这次来了，下次还来的概率"。比如100个新用户，第一个月后还剩60个（留存率60%），第二个月后还剩40个（留存率40%）。留存概率越高，说明用户越忠诚，生意越稳。用留存概率还能预测用户会不会复购，提前做好运营。
生活比喻：就像开饭馆 — 第一次来吃的客人，下次还来吗？常来的回头客越多，说明味道越好，生意越稳定。
概念思维导图
电商实际案例
京东会员体系优化：某店铺做 cohort（同期群）分析，发现2024年1月的新用户，次月留存率只有30%，而行业平均是40%。再细分发现，买过2次以上的用户留存率高达70%，只买1次的只有15%。于是重点做"首单后30天复购激励"，发优惠券、推新品，次月留存率从30%提升到了45%。

  • 衡量忠诚
  • 预测复购
  • 指导运营
  • 计算LTV

项目要求
📊需要数据用户注册时间、首次购买时间、每次购买时间，至少3个月以上的数据
🔧需要工具AI可以帮你分析和解释；Excel做数据透视表；Python更灵活
🎯适用场景用户忠诚度分析、复购预测、会员体系设计、用户生命周期价值计算

基础提示词

用户画像统计（描述统计）
高级方法
什么是用户画像统计？（大白话）
就像给用户"画一张像" — 把用户的各种特征（年龄、性别、消费多少、买什么、多久来一次）统统统计出来，最后拼成一个"典型用户"的样子。用户画像统计就是用描述统计的方法，把用户数据从各个维度总结归纳一下，让你一眼就知道"我的用户都是些什么人"。比如平均年龄28岁、女多男少、平均消费200元、主要买护肤品、每月来一次。
生活比喻：就像警察画嫌疑人画像 — 根据目击者的描述，把身高、体型、脸型、发型都拼起来，最后得到一个清晰的人物形象。
方法思维导图
电商实际案例
淘宝美妆店铺用户画像：某美妆店有50万用户，通过描述统计画出用户画像：78%女性，25-30岁占比最高（35%），主要来自一二线城市（60%），平均客单价180元，月均购买1.5次，最爱买的品类是面膜和口红，对价格敏感（喜欢促销时下单）。基于这个画像，店铺调整了选品策略，主打高性价比日韩美妆，销量涨了25%。

相关性分析（商品关联）
高级方法
什么是相关性分析？（大白话）
两个东西是不是"一起变"？比如天气越热，冰淇淋卖得越多，这就是正相关；价格越贵，买的人越少，这就是负相关。相关性分析就是算两个变量之间"关系有多密切"，用相关系数（-1到1之间）表示。越接近1，正相关越强；越接近-1，负相关越强；接近0就是没关系。电商里常用来找商品关联、找影响销量的关键因素。
生活比喻：就像"身高和体重" — 一般来说个子越高的人越重，这两个就是正相关；但"身高和考试分数"就没啥关系，相关系数接近0。
方法思维导图
电商实际案例
京东销量影响因素分析：某3C店铺想知道什么因素影响销量。算了一下相关系数：价格和销量相关系数-0.7（负相关强，越贵卖越少）、好评率和销量0.6（正相关，好评多卖得多）、广告投放和销量0.5（投广告有用）、客服响应时间和销量-0.3（有点关系但不强）。于是重点做降价和提升好评率，销量果然涨了。
✅ 核心作用：
  • 找关键因素
  • 商品关联
  • 变量筛选
  • 辅助决策
📋 使用前必看：前置条件
项目要求
📊需要数据多个数值型变量的数据（销量、价格、好评率、广告费用等），至少几十组数据
🔧需要工具AI可以帮你算和解释；Excel有CORREL函数；Python用pandas/numpy
🎯适用场景找影响销量的关键因素、商品关联分析、特征筛选、验证业务假设
⚠️注意事项相关不等于因果！两个东西相关不代表一个导致另一个；可能有第三个因素在影响
⏱️预计耗时简单相关分析10-15分钟，深度分析（含可视化和解读）20-30分钟
高级提示词
帮我做电商数据的相关性分析： 1. 先列出所有数值型变量，两两计算相关系数（皮尔逊或斯皮尔曼） 2. 画个相关系数热力图，直观展示各变量之间的关系强弱 3. 找出和目标变量（如销量/转化率）相关性最强的前5个因素 4. 区分正相关和负相关，分别解释一下业务含义 5. 特别提醒：相关不等于因果，分析可能的因果关系和中间变量 6. 基于相关性分析，给运营和选品提3-5条可落地的建议

回归分析（销售预测）
高级方法
什么是回归分析？（大白话）
相关性分析只告诉你"两个东西有关系"，回归分析更进一步，能告诉你"具体是什么关系，一个变了，另一个会变多少"，还能用来预测。比如价格每降10块，销量能涨多少？广告每多投1万块，销售额能增加多少？回归分析就能算出这个"数学公式"，然后你把数代进去就能预测。
生活比喻：就像"身高预测体重" — 医生有个公式，知道你身高就能大概算出标准体重，这就是回归分析的结果。
方法思维导图
电商实际案例
拼多多销售预测：某食品店想预测下个月的销量。用过去12个月的数据做回归分析，发现：广告投入每增加1000元，销量增加50单；价格每降1元，销量增加30单；好评率每升1%，销量增加20单。模型R²=0.85（拟合得不错）。于是下个月投2万广告、降价2元、努力提升好评率，预测销量能涨40%，实际涨了35%，挺准的。
✅ 核心作用：
  • 销量预测
  • 因素量化
  • 效果预估
  • 辅助决策
📋 使用前必看：前置条件
项目要求
📊需要数据历史数据（至少几十组，越多越准），包含因变量（要预测的）和多个自变量
🔧需要工具AI可以帮你建模和解释；Excel有数据分析工具；Python用sklearn/statsmodels
🎯适用场景销售预测、价格弹性分析、广告效果预估、关键因素量化分析
⚠️注意事项线性回归有前提假设（线性、独立、正态、等方差）；别过度拟合；预测别外推太远
⏱️预计耗时简单线性回归15-20分钟，多元回归建模+验证30-60分钟
高级提示词
帮我用回归分析做电商销售预测： 1. 先明确目标：要预测什么（销量/销售额/转化率），有哪些可能的影响因素 2. 做相关性分析初筛，把相关性弱的变量先去掉 3. 用多元线性回归建模，写出回归方程（每个因素的系数是多少） 4. 评估模型效果：R²是多少？每个变量的P值显著吗？ 5. 解释每个因素的影响：价格每变1元，销量变多少？广告每投1万，销量变多少？ 6. 用模型做一个预测场景：假设下个月投X万广告、降价Y元，预测销量能到多少 7. 给提升模型效果提建议（还能加什么变量、用什么更复杂的模型）

A/B测试（效果验证）
高级方法
什么是A/B测试？（大白话）
两个方案哪个好？不能拍脑袋，得用实验说话。A/B测试就是把用户随机分成两组，一组用老方案A，一组用新方案B，然后比两组的效果。比如A组看红色按钮，B组看绿色按钮，看哪个点击率高。关键是用户要"随机分"，而且样本量要够，不然可能是运气好，不是真的好。
生活比喻：就像试菜 — 想知道新菜好不好吃，找两拨人，一拨吃老菜，一拨吃新菜，然后比哪个好评多。
方法思维导图
电商实际案例
淘宝详情页优化A/B测试：某女装店铺主图改了，想知道新主图好不好。做A/B测试：50%用户看老主图（A组），50%看新主图（B组），各10万流量。结果A组点击率3%，B组点击率3.5%。用假设检验一算，P值<0.05，差异显著，不是运气。于是全量换成新主图，整体点击率涨了15%。
✅ 核心作用：
  • 科学验证
  • 数据决策
  • 降低风险
  • 持续优化
📋 使用前必看：前置条件
项目要求
📊需要数据A/B两组的测试数据，包括曝光量、点击量、转化量等，样本量要足够大
🔧需要工具AI可以帮你算显著性和解读；有专门的A/B测试平台；Excel也能算
🎯适用场景页面优化、价格测试、文案测试、活动效果验证、功能迭代效果评估
⚠️注意事项必须随机分组、只能改一个变量、样本量要够、别偷看数据提前结束、注意统计显著性
⏱️预计耗时设计测试15分钟，跑测试看流量大小（几天到几周），分析结果15-20分钟
高级提示词
帮我做电商A/B测试的设计和分析： 1. 先明确测试目标：要测什么？核心评估指标是什么（点击率/转化率/客单价）？ 2. 帮我算一下需要多少样本量才能测出显著差异（预估提升幅度） 3. 设计测试方案：怎么分组？流量比例？测试多长时间？注意什么？ 4. 测试结束后，用假设检验判断两组差异是不是真的显著（P值、置信区间） 5. 除了核心指标，也要看其他指标有没有变差（别只看一个指标） 6. 给出结论：要不要全量上线？为什么？如果效果不好，可能的原因是什么？

时间序列分析（销售趋势）
高级方法
什么是时间序列分析？（大白话）
就是把数据按时间顺序排好，看看它过去怎么走，然后预测未来会怎么走。时间序列分析就是分析"随着时间变化的数据"，找出规律，预测未来。比如去年双11销量涨了多少，今年大概会涨多少？这个月每天的销售额，下个月大概是多少？电商里常用来做销量预测、库存规划、活动预判。
生活比喻：就像看天气预报 — 把过去几天的气温、风向、云图都拿出来，找出规律，然后预测明天是晴是雨、温度多少。
方法思维导图
电商实际案例
拼多多店铺销量预测：某零食店想知道下个月能卖多少，好提前备货。把过去一年的日销量按时间排好，用时间序列分析发现：整体是上升趋势（每月涨5%），周末销量高（比工作日高30%），每月中旬有小高峰（发工资后）。预测下个月大概能卖120万，备了150万的货，结果实际卖了128万，误差很小，没断货也没压太多库存。

假设检验（效果验证）
高级方法
什么是假设检验？（大白话）
就是"用数据证明一个说法对不对"。比如你说"新包装能提升销量"，不能光感觉，得用数据验证。假设检验就是先提出一个假设，然后用数据来判断这个假设成不成立。核心是看差异是"真的有效果"还是"只是运气好"。常用P值来判断，P<0.05就是"大概率不是运气"，叫"统计显著"。
生活比喻：就像法官断案 — 先假设"嫌疑人无罪"（原假设），然后找证据（数据），如果证据足够充分（P值很小），就推翻假设，判有罪（有显著差异）。

电商实际案例
天猫优惠券效果验证：某店发了优惠券，想知道是不是真的提升了转化率。有券组1万人，转化率5%；没券组1万人，转化率4%。看起来涨了1个点，但用假设检验一算，P值=0.02，小于0.05，说明这个差异大概率不是运气，优惠券真的有效。于是决定长期发券，每月多赚20万。

  • 验证效果

  • 降低风险

⚠️注意事项统计显著≠实际重要（样本量大的时候很小的差异也会显著）；别只看P值

高级提示词
帮我做电商数据的假设检验分析： 1. 先明确要验证什么问题？原假设和备择假设分别是什么？ 2. 数据是什么类型？用哪种检验方法合适（T检验/卡方检验/方差分析等）？ 3. 计算检验统计量和P值，解释一下P值是什么意思 4. 用显著性水平（通常0.05）判断：拒绝原假设还是不拒绝？ 5. 除了P值，也算一下效应量（差异到底有多大），避免样本量大导致的"假显著" 6. 给出置信区间，让结果更完整 7. 用大白话总结结论：这个效果是真的还是可能是运气？业务上要不要采纳？

聚类分析（用户分群）
高级方法
什么是聚类分析？（大白话）
就是"物以类聚，人以群分" — 把相似的东西归为一类。聚类分析就是让电脑自动把相似的数据分成几堆，每一堆里的都很像，不同堆的不一样。电商里最常用的就是用户分群：把用户分成"高价值铁粉"、"价格敏感型"、"偶尔来逛逛"等等，然后对不同群用不同的运营策略。
生活比喻：就像整理衣柜 — 把衣服分类：上衣放一堆，裤子放一堆，袜子放一堆。每堆里的东西用途相似，不同堆的完全不一样。聚类就是电脑帮你自动分类，而且是按"相似度"来分的。
方法思维导图
电商实际案例
京东用户分群运营：某家电店有20万用户，用K-Means聚类把用户分成了4群：①"高端土豪"（5%）：客单价5000+，只买高端品牌；②"理性中产"（30%）：客单价2000，比价格也比品质；③"价格敏感"（45%）：只在促销时买，客单价800；④"偶尔逛逛"（20%）：半年买一次，客单价500。然后对每群发不同的文案和优惠券，整体转化率涨了20%。
✅ 核心作用：
  • 用户分层
  • 精准运营
  • 差异化策略
  • 提升效率
📋 使用前必看：前置条件
项目要求
📊需要数据用户/商品的多维度数据（消费、行为、偏好等），样本量越大分群越准
🔧需要工具AI可以帮你做聚类分析和解读；Python用sklearn；SPSS也可以做
🎯适用场景用户分群运营、商品分类、个性化推荐、市场细分、客户价值分层
⚠️注意事项分几类没有标准答案，要结合业务理解；分群后一定要给每个群"贴标签"才有用
⏱️预计耗时简单分群20-30分钟，含画像和运营策略的完整分析45-60分钟
高级提示词
帮我做电商用户聚类分析（用户分群）： 1. 先说说用哪些维度来分群比较合理（消费/行为/偏好等），为什么？ 2. 用K-Means或其他合适的聚类方法帮我分群，确定分几类最合适（肘部法则/轮廓系数） 3. 每类用户有多少人？占比多少？ 4. 给每类用户起个好记的名字，详细描述每群的特征画像 5. 对比不同群的关键指标差异（消费金额、频次、品类偏好等） 6. 针对每类用户，分别给3-5条具体的运营建议（怎么触达、推什么、怎么转化） 7. 说说怎么验证分群效果？后续怎么迭代优化？

协同过滤（推荐算法）
高级方法
什么是协同过滤？（大白话）
就是"人以群分，物以类聚"的推荐逻辑 — 跟你品味相似的人买了什么，就给你推什么；或者你买了A，大家买A的时候也常买B，就给你推B。协同过滤就是利用"集体智慧"来做推荐，不用懂商品本身，只看用户行为。电商的"猜你喜欢"、"买了又买"、"看了又看"，基本都是这个原理。
生活比喻：就像朋友推荐 — 你朋友跟你口味很像，他说"这部电影超好看"，你大概率也会喜欢。协同过滤就是电脑自动找"跟你像的人"，然后把他们喜欢的推荐给你。
方法思维导图
电商实际案例
淘宝"猜你喜欢"推荐：某女装店用协同过滤做推荐。基于用户的购买和浏览记录，找到"品味相似的用户"和"经常一起买的商品"。比如买了连衣裙的人，70%也买了同系列的小外套；喜欢森系风格的用户，也喜欢棉麻材质。给每个用户推个性化商品，推荐位的点击率从2%涨到5%，推荐带来的销售额占比从10%涨到25%。

  • 个性化推荐
  • 提升转化
  • 增加客单价
  • 发现长尾

项目要求
📊需要数据用户行为数据（浏览/收藏/加购/购买），用户和商品数量都不能太少
🔧需要工具AI可以帮你设计推荐逻辑和解读；Python用surprise/sklearn；有专门的推荐系统
🎯适用场景猜你喜欢、关联推荐、个性化首页、商品详情页推荐、邮件/短信推荐

⏱️预计耗时设计推荐逻辑20-30分钟，简单实现30-45分钟，完整系统需要开发资源

帮我设计电商协同过滤推荐系统： 1. 先说说我的业务场景适合哪种推荐（基于用户/基于物品/混合），为什么？ 2. 需要哪些数据？数据格式应该是什么样的？ 3. 怎么计算相似度？用什么指标（余弦相似度/杰卡德/皮尔逊）？ 4. 冷启动问题怎么解决？新用户、新商品刚来的时候推什么？ 5. 怎么评估推荐效果？用什么指标（点击率/转化率/客单价/留存）？ 6. 给我设计一个简单可落地的推荐方案：首页怎么推、详情页怎么推、购物车怎么推 7. 说说怎么A/B测试推荐效果？怎么持续优化？
